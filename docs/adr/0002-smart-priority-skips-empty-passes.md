# Smart priority skips empty priority passes

The rules engine uses "smart priority" — it only stops to ask an agent for a decision when that player actually has a legal instant-speed action available (a castable instant, an activatable ability, a relevant triggered ability). When no player can respond, priority passes are skipped entirely and the top of the stack resolves immediately. We chose this over full priority (stopping at every legal priority point per the Comprehensive Rules) because Commander games involve hundreds of priority passes per turn across 4 players, and >90% of them result in "pass." Full priority would dominate simulation time with trivial no-op decisions. Smart priority is functionally equivalent in outcome — if no player can act, the result of passing is the same as skipping. The tradeoff is a subtle correctness risk: if the "can this player respond?" check has bugs, the engine might skip a priority window where a player should have been able to act. This must be tested carefully against known interaction sequences.
